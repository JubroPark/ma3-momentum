# 통합 투자 전략 모바일 서비스 — 구현 계획/설계안 (v2)

> 하나의 PWA에서 **「나스닥 마삼 대응」**과 **「50일 이평선 스크리너」** 두 전략을 상단 토글로 전환해 보는 모바일 서비스.
> 데이터·인프라(코어)는 공유, 전략별 엔진·화면은 분리.
> ⚠️ 스크리닝/의사결정 보조 도구이며 투자 권유 아님. 실거래 전 백테스트 검증 필수.

## 0. 목표 한 줄 요약
US 일별 EOD 데이터를 매일 1회 배치 처리하는 **공통 코어** 위에, 성격이 다른 두 전략 엔진을 얹고, 모바일에서 **세그먼트 토글**로 전환해 사용한다. 전면 무료 스택(Next.js 14+ PWA / Vercel Hobby / GitHub Actions / yfinance·Stooq / Web Push) 유지.

## 1. 두 전략의 본질 차이 (설계 출발점)

가장 먼저 인정해야 할 점: 두 전략은 단순히 "버튼만 다른 같은 화면"이 아니다. 분석 단위와 산출물 성격이 다르므로 **IA(정보 구조)는 분리**하고 **하부 데이터/연산만 공유**한다.

| 축 | 마삼 대응 (전략 A) | MA50 스크리너 (전략 B) |
|---|---|---|
| 분석 단위 | 시장(나스닥 지수) + 1등주 1개 | 개별 종목 다수 (워치/보유) |
| 성격 | 의사결정 플레이북 = **국면 모니터 + 행동 가이드** | 종목 스크리너 = **시그널 생성기** |
| 입력 | 자동(지수/금리) — **입력 불필요** | 자동(워치·보유 종목) + 포지션 |
| 출력 | 현재 국면 + 권고 행동 + 트리거 알림 | 매수후보 랭킹 + 보유 관리 시그널 |
| 자동화 | **완전 자동** (포지션 추적 미포함) | 거의 완전 자동 |
| 화면 수 | 단일 상태 중심 (대시보드형) | 다종목 리스트 중심 (스크리너형) |

> 핵심 결론: A는 "지금 룰북상 어느 국면이고 뭘 해야 하나"를 알려주는 **단일 상태 대시보드**, B는 "어느 종목이 매수/매도 후보인가"를 뽑는 **다종목 스크리너**. 공유하는 건 데이터·연산·인프라·알림·차트뿐이다.

## 3. 공통 코어 (Shared Core)

### 3-1. 데이터 레이어 (US · 무료)

두 전략의 데이터 요구를 합치면 한 번의 수집으로 둘 다 커버된다.

| 데이터 | 소스(무료) | 쓰임 |
|---|---|---|
| 일봉 EOD (수정주가) | Stooq(CSV·무키) / yfinance | 공통 |
| 나스닥 지수 | `^IXIC` **[확정]** | 마삼 감지·전고점·V자 (단일 기준) |
| S&P500 / 벤치마크 | `^GSPC` / SPY | MA50 레짐·RS |
| 1등주 시총 (mcap) | yfinance `ticker.info['marketCap']` — 대형주 상위 10개 | 마삼 1등주 자동 판정 · 1등−2등 시총차 |
| 참조 유니버스 | S&P500 + NDX 종가 롤링 캐시 | MA50 RS percentile |
| 섹터 | SPDR 11개 ETF | MA50 섹터 RS |
| 금리 | FRED `DFF`, 10Y | **제로/비제로 분기(마삼 V자 임계값)** + 레짐 참고 |
| VIX | `^VIX` | MA50 레짐 참고 / 마삼 동적기준 옵션 |
| 헤지 자산 4종 | yfinance: `TLT`·`IAU`·`GLD`·`TIP` 일봉 | 금리환경별 매도물량 배치처 안내 |
| Fear & Greed | CNN 비공식 API · 마지막 성공값 24h 캐시 · 실패 시 N/A | 시장환경 보조 패널 |
| RSI / MFI | 1등주 종가·거래량으로 자체 계산 (RSI14·MFI14) | 추가 자금 투입 타이밍 판단 |
| PMI (제조업·서비스업) | FRED API `NAPM`(ISM 제조업) · `NMFCI`(ISM 서비스업) · 월 1회 자동 갱신 | 시장환경 (보조 참고) |
| 펀더멘털 | 사용자 체크리스트 | MA50 보유 판단 |

> 참조 유니버스는 **종가만 롤링 캐시**(전체 히스토리 매일 재수집 X) → 무료 한도 유지.

### 3-2. 연산 파이프라인 (GitHub Actions cron · 18:00 ET)

```
1. EOD 수집: 지수(`^IXIC`·`^GSPC`·`^VIX`) + 1등주 mcap(상위 10개) + 헤지자산(`TLT`·`IAU`·`GLD`·`TIP`) + 워치/보유 + 참조유니버스 + SPY + 섹터 + FRED(`DFF`·`DGS10`·`NAPM`·`NMFCI`) + F&G
2. 공통 지표: MA50/MA200 · VolMA · ATR14 · slope · gap · highN · regime · 금리환경
3-A. 마삼 엔진 평가  → masam.json
3-B. MA50 엔진 평가  → signals.json / positions.json / transitions.json
4. 전일 스냅샷 대조 → 신규 트리거 추출
5. 트리거 → Web Push (VAPID, 무료)
6. 저장: repo JSON / Gist / Supabase·Upstash free
[PWA] 정적 JSON fetch → 렌더
```

### 3-3. 저장 / 알림 / PWA 셸
- **저장(무료)**: repo JSON 또는 Gist(공개 신호) + Supabase/Upstash free(사용자별 워치·포지션·설정).
- **알림**: Web Push/VAPID. 알림에 전략 배지(`[마삼]` / `[MA50]`) 부착, 통합 알림 센터에서 필터.
- **PWA 셸**: Next.js 14 App Router · TS · Tailwind · lightweight-charts · 정적 JSON 오프라인 캐시 · 설치형.

| 재진입 분할 | 35% / 35% / 30% 3트랜치<br>−1~−2% 하락일: 하락일 ×3 / −2% 이상 폭락: 당일 3분할 | ✅ |
| 추가 자금 투입 조건 | RSI14 ≤ 50 **AND** MFI14 ≤ 50 동시 충족 | ✅ |

### 4-2. 마삼 상태 머신 (3-Mode)

마삼 모드는 이제 **3가지 모드**로 동작한다. 모든 전환은 데이터 자동 판정.

```
MODE 1 : REBALANCING  ─── 마삼 없음 + 1등주 전고점 대비 하락
MODE 2 : CRISIS        ─── 나스닥 -3% 발생 (위기)
MODE 3 : PANIC         ─── 같은 달 마삼 4회↑ (공황 확정)
```

**MODE 1 — REBALANCING (평상시 리밸런싱)**
```
1등주 전고점 대비 하락 감지 (마삼 無)
  → -2.5% 하락마다 1등주 10% 매도 (현금화)
  → 최대 한도: 25% or 50% (사용자 설정 ⑪)
  → 개별 종목 이슈면 → QQQ 10% 매수
  → 시장 전체 위기면 → 현금 보유 → 마삼 발생 시 MODE 2 전환

  ✦ 비중 카드: 현재 현금화 진행도 (0%→최대한도)
  ✦ 반등 시: 최종 구간에서 2구간 상승 → 주식 전량 재매수
    (기준: 비양적완화 +5% / 양적완화 +2.5%)
```

**MODE 2 — CRISIS (위기 · 말뚝박기)**
```
마삼 발생
  → [수익 중] 말뚝 비중 제외하고 나머지 전량 매도
  → [손실 중] 전량 매도 후 말뚝 재진입
  ※ 수익/손실 판단 불가 → 조건부 안내 카드 2종 표시

  말뚝박기 비중 자동 계산:
  ┌─ 비제로금리(DFF > 0.25%): 초기 50%, -5% 하락마다 +10%
  └─ 제로금리(DFF ≤ 0.25%):  초기 25%, -2.5% 하락마다 +10%
  MAX: 100% (V자 2구간 or 고점 -30% or 올인조건 ①~④)

  헤지 자산 배치 (매도 현금 → 아래로):
  ┌─ 비제로금리 + QE이전  → TLT
  ├─ 제로금리 / QE시작    → IAU+GLD / TIP (1:1)
  └─ 방향성 불명          → 달러(현금)

  ✦ 비중 카드: 주식 X% / 헤지 Y% / 현금 Z% (실시간 계산)
  ✦ 다음 트리거까지 거리 표시:
    "V자 올인까지 +4.7% 필요 / 긴급올인까지 -11.8% 필요"
```

**MODE 3 — PANIC (공황)**
```
공황 확정: 같은 달 마삼 4회↑
  └─ + 전년도 ^IXIC 역년 ≥ +45% → PANIC_EMERGENCY 등급 추가

──────────────────────────────────────
[PANIC_EMERGENCY] 공황 비상 매뉴얼 적용
──────────────────────────────────────
  말뚝박기 즉시 중단 → 전량 매도 → 현금 100%
  ❌ 고점 −30% 도달해도 몰빵 없음  ← 평시와 핵심 차이
  ❌ 1등주 −30% 하락해도 몰빵 없음
  V자 올인 기준 강화:
    비제로금리: 6구간 = +30% (평시 2구간 +10%의 3배)
    제로금리:   6구간 = +15% (평시 2구간 +5%의 3배)
  → 6구간 달성 or 기타 해제조건 충족까지 현금 100% 대기

──────────────────────────────────
[PANIC_BASIC] 일반 공황 매뉴얼 적용
──────────────────────────────────
  (비상 등급 미해당 시, 레거시 매뉴얼 기준)
  TLT / IAU+GLD+TIP 헤지 운용 유지

──────────────────────────────────
헤지 자산 운용 (공황 공통)
──────────────────────────────────
  TLT 보유 중:
    → 공황 초반에만 유효 (급등 후 급락 위험)
    → Exit: Fed 금리인하/양적완화 조치 OR
             매수세력 차익실현 징후 OR
             중앙은행 미국채 매도 감지
    → TLT 매도 후 → 현금(달러) 보유

  IAU/GLD/TIP 보유 중:
    → 양적완화 시에만 유효
    → Exit: Fed 금리인상 or 양적완화 축소
    → 전량 매도 → 현금(달러) 보유

  환율 급등 (원화 약세 / 달러 강세):
    → 달러 → 원화 전환 알림
    → 이머징마켓 환율 안정화 후 → 원화 → 달러 복귀 알림

──────────────────────────────────
공황 해제 & 재진입
──────────────────────────────────
  조기 진입:
    ① 8거래일 연속 상승 → 8일째 종가 매수
    ② 나스닥 전고점 돌파 → 당일 종가 매수 (공황 완전 해제 아님)

  안전 진입:
    ③ 마지막 마삼일 +2개월+1일 → 당일 종가 매수

  재진입 분할 (3트랜치):
    종목 −1~−2% 하락일: 하락일 ×3회에 걸쳐 각 33%씩 = 35/35/30
    종목 −2% 이상 폭락: 당일 3분할 = 35/35/30

  ✦ 비중 카드: 주식 0% / 헤지 Y% / 현금 Z%
  ✦ 재진입 시: 1트랜치(35%) → 2트랜치(35%) → 3트랜치(30%) 단계 표시
  ✦ 추가 자금 투입: RSI14 ≤ 50 AND MFI14 ≤ 50 동시 충족 시 배지 표시
```

## 5. 전략 B — MA50 스크리너 엔진

> 소스: `모멘텀_스크리너.md` v3 전체 반영.
> US 전용 · 일별 EOD · 데이터·인프라는 §3 공통 코어 공유.
> ⚠️ 스크리닝 보조 도구, 투자 권유 아님. 실거래 전 백테스트 검증 필수.

### 5-1. 지표 정의

표기: `close[0]`=오늘. 임계값은 전부 파라미터(§5-8).

- `MA50`, `MA200`, `VolMA20`, `VolMA3`
- `MA50_slope_pct` = `(MA50[0]−MA50[10])/MA50[10]` (또는 회귀 기울기)
- `gap50` = `(close−MA50)/MA50`, `gap200` = `(MA50−MA200)/MA200`
- `highN` = `max(high[1..breakout_high_lookback])`
- `ATR14` (사이징·이상치용)
- **RS (상대강도):**
  - `RS_raw` = `0.5·(r20ₛ−r20ₘ) + 0.3·(r60ₛ−r60ₘ) + 0.2·(r120ₛ−r120ₘ)` (vs SPY)
  - **`RS_pct`** = 참조 유니버스(S&P500+NDX) 내 `RS_raw`의 percentile rank (0~100) ← 주 지표
  - `RS_z` = 참조 유니버스 z-score (옵션). 워치리스트만으론 산출 금지 (표본 부족).
  - `RS_sector_pct` = 섹터 ETF 기준 동일.
  - ※ 참조 유니버스는 일별 종가만 롤링 캐시(전체 히스토리 매일 재수집 X) → 무료 한도 유지.

---

### 5-2. 레짐 필터

```
regime = (SPY_close > SPY_MA200) ? RISK_ON : RISK_OFF
```

| 구분 | RISK_OFF 동작 |
|---|---|
| 매수 strict (기본) | BUY 전면 비활성 |
| 매수 soft (옵션) | `vol_mult↑`, `overshoot_max↓`, Track B만 허용 |
| **청산 강화** | `max_hold_watch_days` 단축(예 3) · `consec_below_sell` 축소(예 1) |

---

### 5-3. 매수 — 2트랙 + 반등

공통 게이트: 레짐 통과, `gap50 ≤ overshoot_max`, `volume[0] ≥ vol_mult·VolMA20` (hard).

**Track B — 강돌파 (STRONG_BREAKOUT, 기본):**
```
선행: close[1] ≤ MA50[1] AND 최근 lookback_below일 close<MA50 ≥ 60%
돌파: close[0] > MA50[0]  AND  close[0] > highN   # N일 신고가 (fake breakout 차단)
```

**Track A — 초기추세 (EARLY_TREND, 옵션·공격형):**
```
돌파: close[0] > MA50[0] AND close[1] ≤ MA50[1]   # 신선한 50선 크로스 (highN 미충족 허용)
안전장치(필수): RS_pct ≥ rs_early_th(예 80)  AND  MA50_slope_pct > 0
```
> A는 B보다 이르지만 위험 — RS 상위·상승기울기·거래량으로 보완해 가짜돌파 회귀 방지. `signal_type`으로 구분.

**Track Bounce — 50선 지지 후 반등 (BOUNCE):**
```
추세: 최근~20일 close>MA50 다수 AND slope_pct>0 AND MA50>MA200
근접: min(low[0..k]) ≤ MA50·(1+proximity_pct)
지지: min(close[0..k]) ≥ MA50·(1−break_tol)
반등: close[0]>close[1] AND close[0]>MA50
```
거래량 secondary(`VolMA3[0]>VolMA3[1]`)는 **옵션 플래그 + score 가산** (hard filter 아님).

---

### 5-4. 매도 — 3-State

`breakdown = close[0] < MA50·(1−sell_tol) OR (close[0]<MA50 AND close[1]<MA50)`

**상태 결정 (위→아래 우선순위):**

| # | 조건 | 결과 |
|---|---|---|
| 1 | `close ≥ MA50` | **OK** (리셋) |
| 2 | `close<MA50` 연속 `consec_below_sell`일 | **SELL (강제)** |
| 3 | HOLD-WATCH `days_in_state > max_hold_watch_days` | **SELL (강제)** |
| 4 | `breakdown` AND `close < MA200` | **SELL** |
| 5 | `breakdown` AND `close>MA200` AND `RS_pct>50` | **HOLD-WATCH** |
| 6 | `breakdown` (그 외) | **SELL** |

- HOLD 진입 판정은 #5로 단순·설명가능.
- **원칙 보존(삭제 X)**: 이격수렴(`gap200`)·섹터 모멘텀·확신태그는 **score 가중 + UI 컨텍스트**로 보존 → HOLD/SELL 랭킹에 반영 (확신태그+이격수렴이면 같은 HOLD라도 상단 노출).
- `consec_below_sell`·`max_hold_watch_days`는 RISK_OFF에서 자동 축소.

### 5-5. 스크리닝 상태 머신 (ticker 단위)

```
WATCH → BUY → HOLDING → SELL-WATCH → SELL → WATCH
```
저장: `state, prev_state, entered_at, days_in_state, last_signal_date`.
전이 시 `days_in_state = 0`. 전이 기록은 `transitions.json`에 append-only.

### 5-7. 스코어 공식

각 컴포넌트 0~1 정규화 → 가중합 ×100.

```
vol_score      = min(vol_ratio / vol_cap, 1)              # vol_ratio=volume/VolMA20
breakout_score = min(gap50 / overshoot_max, 1)             # 방어선 위 돌파폭
rs_score       = RS_pct / 100
trend_score    = 0.5·min(MA50_slope_pct / slope_cap, 1) + 0.5·(MA50>MA200 ? 1 : 0)

score = 100 · ( w1·vol_score + w2·breakout_score + w3·rs_score + w4·trend_score )
보너스(옵션): VolMA3 상승 +b, 이격수렴+확신태그 +c    # HOLD 랭킹용
```
기본 가중 `w1..w4 = 0.25` (튜닝 대상), `slope_cap = 0.05`.

| 파라미터 | 기본값 | 설명 |

| `volconfirm` | base | base / confirmed |

### 5-9. 백테스트 & 검증

- **생존편향 제거**: point-in-time 유니버스 (상폐·교체 포함).
- **체결**: 시그널=종가 산출 → **익일 시가 체결** (look-ahead 방지) + 슬리피지(bps) + 커미션.
- **포지션 사이징**: fixed-fractional(`risk_per_trade`) 기본, ATR 기반(`atr_mult`) 옵션 — 미적용 시 성과 왜곡 주의.
- **데이터 정합성**: 수정주가 분할·배당 검증. **이상치(±20%/일)는 자동폐기 X → 검수 플래그** 후 코퍼레이트 액션 대조 (실적 갭·정상 급등 오탐 방지).
- 성과 지표: 승률·평균손익·R-multiple·MDD·노출기간·휩쏘율.
- 튜닝: 그리드/베이지안 + **워크포워드** (인/아웃샘플 분리).

**앱 셸:** 상단 세그먼트 토글 `[마삼 대응] [MA50 스크리너]` — 선택 유지, 알림 센터 공유.

**마삼 대응 모드 (대시보드형, 3탭):**

1. **현재 국면**
   - 모드 배지: `REBALANCING` / `CRISIS` / `PANIC`
   - 비중 카드 (전 모드 상시 표시): `주식 X% / 헤지 Y% / 현금 Z%`
   - 다음 트리거까지 거리: "V자 올인 +4.7% 필요 / 긴급올인 -11.8% 필요"

2. **행동 가이드** (모드별 다른 내용)
   - REBALANCING: 현금화 진행 단계 + 반등 재매수 기준
   - CRISIS: 올인 조건 ①~⑤ 체크리스트(등급 배지) + 헤지 자산 배치 안내 + 조건부 안내 카드 (수익 중 / 손실 중)
   - PANIC: 공황 유형 배지 (`BASIC` / `EMERGENCY 🔴`) + 해제 조건 카운트다운 + 헤지 자산 exit 감시 상태 + 환율 알림 + 재진입 35/35/30 단계 트래커<br>⚠️ EMERGENCY 시: "고점 -30% 몰빵 없음 · V자 6구간 기준 적용 중" 경고 배너 표시

3. **시장환경/설정**
   - 금리(DFF·방향성)·QE 여부·VIX·F&G·PMI
   - 헤지 자산 현재가 (TLT·IAU·GLD·TIP)
   - 리밸런싱 한도 토글: `25%` / `50%` (⑪ 결정)
   - 옵션 토글 (보완 파트 B·C·D)

**MA50 스크리너 모드 (스크리너형, 4탭):**
1. **매수 후보** (점수순 · STRONG/EARLY/BOUNCE 배지 · RISK_OFF 배너)
2. **보유 관리** (포지션 단위 상태칩·경과일·손익·손절선)
3. **시장 환경** (regime·연준·PMI·VIX·F&G) ← **마삼 모드 탭 3과 컴포넌트 공유**
4. **종목 상세/설정** (차트·펀더멘털 체크리스트·파라미터·전이기록)

> "시장 환경"은 두 전략 공통이므로 **한 컴포넌트로 공유**해 양쪽에서 동일하게 노출. 차트는 lightweight-charts 통일.

## 10. 즉시 다음 액션 (추천 1개)

**Phase 0 공통 데이터 PoC 스크립트 하나**로 두 전략의 핵심 입력을 동시에 검증하는 것을 권장한다. 한 번의 수집으로:
- 마삼 측: 나스닥 일간수익률 −3% 감지 · FRED 금리환경 분기 · 전고점 대비 낙폭
- MA50 측: 참조 유니버스 RS percentile · N일 신고가 · MA50/MA200

이 한 스크립트가 두 전략 **공통의 최대 리스크(무료 데이터로 정확히 산출되는가)**를 동시에 해소하므로, 백테스트보다 먼저 진행하는 것이 비용 대비 효율이 높다.
